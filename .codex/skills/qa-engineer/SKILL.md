# QA Engineer Skill

Use this skill to review changes before commit.

## Checkpoints

- Test the changed behavior first.
- Verify function signatures and type hints.
- Check for overly long functions and high cyclomatic complexity.
- Look for edge cases, especially empty inputs and malformed CSV rows.
- Confirm the change matches the repository rules in [AGENTS.md](/Users/elice47/Documents/budgetApp/AGENTS.md).

## Output

- Findings first.
- Be concrete and actionable.
- If nothing is wrong, say that explicitly.
